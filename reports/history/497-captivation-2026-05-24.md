## Block F — Interview Prep

**If you decide to apply (score >= 3.0):**

### Story 1: Systems Optimization Under Constraints (Mini-Stockfish)
- **Situation:** Building a chess engine neural network trained on 316M examples; evaluation latency was critical bottleneck (1.1ms per move).
- **Task:** Reduce latency to <1ms for real-time gameplay.
- **Action:** Profiled the C++ move generation using flame graphs. Identified two hot loops: bitboard manipulation and move sorting. Rewrote in Cython with C++ backend, removing Python overhead. Added memoization for transposition tables.
- **Result:** Achieved 77.2% latency reduction (1.1ms → 0.25ms). Engine now evaluates 4,400 leaf nodes/sec on commodity hardware.
- **Reflection:** Low-level optimization requires understanding both high-level algorithms (alpha-beta search) and low-level hardware (cache locality, branch prediction). HPC is similar: big picture architecture + detailed optimization.

### Story 2: Learned Unfamiliar Hardware Stack Quickly (FlipperZillow)
- **Situation:** 24-hour hackathon. Task: process real estate images through GPU pipeline. Only available infrastructure was AMD ROCm (unfamiliar; most experience is NVIDIA CUDA).
- **Task:** Set up unfamiliar GPU environment and integrate 5+ pre-trained models.
- **Action:** Spent 2 hours reading ROCm docs and migration guides. Built compatible pipeline. When SAM3D broke on ROCm (dependency conflict), pivoted to NVIDIA cloud instead of debugging forever.
- **Result:** Shipped working 3D house tours with professional AI narration in 24 hours.
- **Reflection:** Pragmatism > perfection. You learn new hardware stacks fast by building, not reading docs. This is exactly what HPC work requires.

### Story 3: Advocated for Technical Architecture (PatentIQ)
- **Situation:** PatentIQ needs vector database for semantic patent search. PMs suggested MongoDB + Pinecone.
- **Task:** Choose the right persistence layer for MVP speed.
- **Action:** Argued for PostgreSQL + pgvector instead. Cheaper, open-source, faster iteration. Convinced data scientist and backend engineer to validate independently.
- **Result:** Got buy-in. Shipped MVP with lower cost and faster iteration.
- **Reflection:** Good engineering is about trade-offs, not silver bullets. In HPC, you're constantly optimizing: latency vs. throughput, memory vs. compute, correctness vs. speed.
