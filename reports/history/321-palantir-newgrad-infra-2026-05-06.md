## Block F: Interview Prep

**5 STAR+R Stories:**

### 1. **Architected Scalable System Under Deadline (PatentIQ MVP)**
- **Situation:** PatentIQ accelerator, 6 weeks, build MVP from scratch with 14-person team.
- **Task:** Design system architecture, make technology decisions, coordinate teams, ship.
- **Action:** I proposed stack (FastAPI + Next.js + Supabase + PostgreSQL pgvector). Made tradeoff decisions (pgvector over Pinecone for cost/control, Redis caching for latency, Auth0 for secure auth). Designed API contracts, reviewed code, unblocked PMs and designers. Adapted architecture as requirements changed.
- **Result:** Shipped working MVP in 6 weeks. System designed for future scaling (horizontal scaling, database partitioning considered).
- **Reflection:** Infrastructure teams make similar tradeoff decisions at larger scale. I understand: control vs. managed services, consistency vs. latency, simplicity vs. optimization.

### 2. **Optimized Latency-Critical System (Groundwork Books + Mini-Stockfish)**
- **Situation:** Two projects with tight latency requirements: (1) e-commerce inventory lookups (2K-5K concurrent users), (2) chess engine AI evaluation (billions of calls during search).
- **Task:** Identify bottlenecks and optimize without sacrificing correctness.
- **Action:** For e-commerce, I profiled the system, identified N+1 problem (repeated Square API calls), implemented two-tier caching (Redis + request coalescing). For chess engine, I profiled critical path, identified move generation bottleneck (40% of latency), rewrote in C++, optimized with Zobrist hashing and SIMD.
- **Result:** Groundwork (2.1s → 380ms p99 latency), chess engine (1.1ms → 0.25ms per eval, 77% improvement).
- **Reflection:** Infrastructure teams care deeply about latency. Palantir's systems serve government agencies where milliseconds matter. Understanding profiling, caching, and optimization is crucial.

### 3. **Understood System Reliability & Consistency Tradeoffs (PatentIQ Cache Design)**
- **Situation:** PatentIQ semantic search needed to balance freshness (correct data) with performance (fast lookups).
- **Task:** Design caching strategy for patent embeddings.
- **Action:** I proposed 2-minute TTL cache (acceptable staleness for patent search, huge latency gains). Implemented with explicit cache invalidation on patent updates. Monitored cache hit rate (90%+). Documented tradeoff: "We accept 2-minute staleness for 90% faster queries."
- **Result:** System performed well, stakeholders understood the tradeoff.
- **Reflection:** Mission-critical infrastructure (Palantir) requires explicit tradeoff analysis. You can't just optimize for speed. You understand consistency, availability, durability (CAP theorem concepts).

### 4. **Led Cross-Functional Deployment (PatentIQ MVP Launch)**
- **Situation:** PatentIQ MVP ready to deploy. Multiple teams dependent on launch: founders (business), PMs (user testing), designers (feature validation).
- **Task:** Coordinate deployment, handle issues, communicate status.
- **Action:** I worked with ops/DevOps (Vercel deployment), frontend/backend (API integration), PMs (timeline). Tested thoroughly before launch. Monitored errors post-launch, rolled out hotfixes. Communicated clearly with non-technical stakeholders about what was working and what wasn't.
- **Result:** Smooth launch, no critical issues, teams could move forward with user research.
- **Reflection:** Palantir's Production Infrastructure team deploys systems to government agencies. Reliability and communication matter. This story shows I understand deployment rigor.

### 5. **Debugged Complex System Under Pressure (FlipperZillow Hackathon)**
- **Situation:** 24-hour hackathon. GPU infrastructure (AMD ROCm) + multiple ML models (DepthAnythingV2, DFormerV2, SAM3D) + Vision Pro integration.
- **Task:** Get system working in 24 hours, handling tool incompatibilities and failures.
- **Action:** I debugged GPU compatibility issues (tool versions, library conflicts), prioritized ruthlessly (cut SAM3D when it wouldn't work on ROCm), integrated 4 working models into pipeline, shipped.
- **Result:** Working system, hackathon finalist.
- **Reflection:** In production infrastructure, systems are complex and failures happen. You debug pragmatically, prioritize ruthlessly, communicate. This story shows resilience.
