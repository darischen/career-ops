Instead of debugging (would waste hours), cut SAM3D, shipped working depth estimation + 3D object extraction using existing models. Coordinated with frontend team to integrate Three.js 3D viewer.
- **Result:** Shipped working 3D house tours with AI narration in 48 hours. Learned 5 ML frameworks, debugged GPU compatibility, shipped pragmatically.
- **Reflection:** Pragmatism wins. I learned that shipping a working product under pressure is better than chasing perfection. At Old Mission, I'd apply that: cut scope, ship, iterate.

#### 2. **Advocated for Technical Approach (and Won)**
- **Story:** PatentIQ — semantic search architecture decision
- **Situation:** PatentIQ MVP needed to search 100K+ patents by semantic similarity (not keyword match). Team debated: Pinecone (managed vector DB) vs. PostgreSQL + pgvector (open-source).
- **Task:** Recommend architecture that balances speed (4-month accelerator), cost (early-stage startup), and iteration velocity.
- **Action:** (You) Analyzed both options. Pinecone: $0.40 per 1M queries, vendor lock-in, slower iteration. PostgreSQL + pgvector: free, portable, integrate with existing Supabase. Proposed pgvector + OpenAI embeddings. Ran cost analysis: pgvector saves 30% vs Pinecone, no vendor lock. Built POC in 2 days to prove it works. Presented to team.
- **Result:** Approved. Shipped PatentIQ semantic search on pgvector. Saved $30K+ cost, faster iteration, no regrets.
- **Reflection:** Technical decisions aren't just about tech. They're about risk, cost, timeline, learning. I learned to advocate with data, not just preference.

#### 3. **Shipped End-to-End Under Constraints**
- **Story:** PatentIQ MVP — full-stack ownership
- **Situation:** 4-month accelerator. Build AI patent search MVP for early-stage founders. 14-person team (PMs, designers, engineers). No clear playbook.
- **Task:** Architect full stack (frontend, backend, database), serve as primary contributor, lead team.
- **Action:** (You) Built FastAPI backend + semantic search + Auth0 + PostgreSQL pgvector. Built Next.js frontend with form validation, patent display, drafting UI. Deployed to Vercel + EC2. Coordinated with PMs on feature prioritization, with designers on UI, with other engineers on testing. Shipped MVP in 6 weeks.
- **Result:** PatentIQ now used by early-stage founders for professional-level patent research. 100+ patents searchable, instant draft generation.
- **Reflection:** End-to-end ownership means managing trade-offs. I learned to say "no" to features that don't matter, "yes" to what users actually need, and lean on teammates for strength.

#### 4. **Technical Leadership (Without the Title)**
- **Story:** PatentIQ team lead role
- **Situation:** 14-person cross-functional team. Junior engineers needed guidance on code standards, testing, design philosophy.
- **Task:** Set expectations for code quality without micromanaging. Build culture of learning, not blame.
- **Action:** (You) Ran code reviews focused on teaching, not gatekeeping. Explained why caching matters (we reduced API latency 90%), why we validate JSON (catches bugs early). Helped junior engineers debug their own issues. Wrote design docs for complex features.
- **Result:** Team shipped faster. Junior engineers leveled up. Code quality improved.
- **Reflection:** Leadership is about making others better, not proving you're smartest in the room.

#### 5. **Technical Depth — Performance Optimization**
- **Story:** Groundwork Books — caching strategy
- **Situation:** E-commerce site with 4,000+ SKUs. Real-time inventory sync with Square API. Page loads were 3-4 seconds (slow). Many users checking inventory concurrently.
- **Task:** Reduce latency. Inventory data is live; can't cache forever.
- **Action:** (You) Designed two-tier caching: (1) Redis cache for frequently-checked SKUs, 5-minute TTL; (2) Client-side request coalescing, batching 50+ concurrent requests into 1 API call. Implemented backend hook to detect "hot" SKUs (sold frequently) and prioritize cache. Result: Square API latency dropped 90% (sub-200ms p99). FCP <0.8s globally.
- **Result:** Page loads <0.8s. Users happy. Server load cut by 70%.
- **Reflection:** Performance is a feature. Users feel the difference. Good engineering is invisible to users but critical to trust.
