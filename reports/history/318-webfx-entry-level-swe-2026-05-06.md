## Block F: Interview Prep

**5 STAR+R Stories:**

### 1. **Owned Full-Stack Feature from Ideation to Production (PatentIQ Semantic Search)**
- **Situation:** PatentIQ needed semantic search over 12K+ USPTO patents. No prior setup; me as lead engineer.
- **Task:** Design, build, and optimize search layer that returns relevant patents in <200ms.
- **Action:** I evaluated options (Pinecone vs. pgvector). Chose pgvector (cost + control). Designed hybrid retrieval (BM25 keyword + OpenAI embeddings). Implemented recursive query builder (Zod validation, custom filters). Built LRU cache (2-minute TTL). Tested edge cases (embedding mismatches, filter combinations, concurrent queries).
- **Result:** Sub-200ms latency, saved $20K/year, system scaled smoothly.
- **Reflection:** Full-stack isn't just front-end + back-end. It's architecture + implementation + optimization + testing.

### 2. **Debugged Production Latency Issue (Groundwork Books)**
- **Situation:** Groundwork Books e-commerce site was slow. Square API calls taking 2+ seconds per inventory lookup. UX suffering.
- **Task:** Identify bottleneck and fix.
- **Action:** I profiled the calls, identified N+1 problem (multiple independent Square API calls for 50+ concurrent inventory lookups). Implemented two-tier caching: Redis (90% reduction to sub-200ms) + client-side request coalescing (HTTP request batching with 2-minute TTL). Reduced p99 latency from 2.1s to 380ms.
- **Result:** Site now feels fast. Repeat visitors no longer experience lag.
- **Reflection:** Optimization isn't glamorous, but it's table stakes. You ship features, you optimize, you monitor, you iterate.

### 3. **Learned New Tech Stack Fast (FlipperZillow, 24-hour hackathon)**
- **Situation:** Hackathon challenge. Tech stack: AMD ROCm (unfamiliar), DepthAnythingV2, DFormerV2, SAM3D, Vision Pro.
- **Task:** Learn, build, ship 3D house tour app in 24 hours.
- **Action:** I dug into ROCm documentation, debugged GPU compatibility (tool stack incompatibilities), prioritized ruthlessly (dropped SAM3D when it broke), integrated 4 models into coherent pipeline.
- **Result:** Working app, hackathon finalist.
- **Reflection:** Learning new tech is 80% reading docs + Stack Overflow, 20% intuition. I'm not afraid of it.

### 4. **Led Cross-Functional Feature (PatentIQ: UI + API + Database Design)**
- **Situation:** PMs wanted patent drafting feature. Required new API, new database schema, new UI screens. Tight timeline.
- **Task:** Unblock multiple teams (backend, frontend, PMs) by clarifying technical approach.
- **Action:** I sketched the API (request/response schemas), proposed database schema (normalization vs. denormalization tradeoff), explained to frontend team (what we can deliver, what's out of scope). Reviewed PRs from other engineers. Adjusted scope when timeline pressured.
- **Result:** Feature shipped on time. No scope creep, no technical regrets.
- **Reflection:** Full-stack includes soft skills: communication, prioritization, and knowing when to say "not in this sprint."

### 5. **Shipped Feature End-to-End in One Sprint (Simplerfy Form-Filling)**
- **Situation:** Simplerfy needed Chrome extension to auto-fill job application forms across 15+ job boards.
- **Task:** Design extension architecture, implement form detection, implement form-filling logic, handle edge cases.
- **Action:** I used Chrome Extension APIs (content scripts, popup UI), built form-detection logic (DOM parsing, field inference), integrated Claude API for smart answers. Tested on Lever, Greenhouse, Handshake, custom job boards.
- **Result:** Extension works on 15+ boards, saves candidates 30+ minutes per application.
- **Reflection:** Sometimes you move fast and aren't perfect. But shipping beats perfect-but-late.
