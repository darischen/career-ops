### Story 5: Production Performance & Pragmatism
**STAR+R:** Groundwork Books inventory caching
- **Situation:** E-commerce site needed real-time inventory from Square API. Naive approach: fetch on every product view. That would exceed API rate limits and be slow.
- **Task:** Reduce latency for 4000+ SKU catalog with live inventory.
- **Action:** Built two-tier cache: Redis (persistent, 90% latency reduction) + client-side request coalescing (batching 50+ concurrent lookups into one API call). Measured p99 latency before and after.
- **Result:** Square API latency reduced by 90%, sub-200ms p99, enabled real-time inventory.
- **Reflection:** Performance isn't about the fanciest algorithm. It's about understanding bottlenecks (API calls, network round-trips) and removing them systematically.

**Why it matters:** Feed ranking at scale has similar constraints. You'll optimize for latency, accuracy, freshness. This story shows you think about real systems.
