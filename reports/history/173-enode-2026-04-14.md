### Story 2: Integration Architecture Under Pressure
**STAR+R:** Groundwork Books API integration (7-month project, real users)
- **Situation:** E-commerce site needed real-time inventory sync from Square API. Square has strict rate limits and latency requirements.
- **Task:** Design integration that doesn't exceed rate limits but keeps inventory fresh.
- **Action:** Built two-tier cache strategy: Redis (persistent storage, 90% latency reduction) + client-side request coalescing (batching 50+ concurrent inventory lookups).
- **Result:** Managed 4000+ SKUs with sub-200ms p99 latency. Real users experienced instant inventory updates.
- **Reflection:** API integration is about constraints. Rate limits, latency budgets, data freshness all trade off. You design for the real constraints, not the ideal case.
