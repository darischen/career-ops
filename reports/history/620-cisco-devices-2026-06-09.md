## Block F — Interview Prep

**Situation 1: Hardware-Software Integration (HB100 Phased Array Radar)**

**Situation:** Build microwave phased array sensor. Need analog frontend + digital signal processing + firmware coordination.

**Task:** Integrate analog output (variable gain op-amp cascade) with digital ADC sampling and FFT-based direction finding.

**Action:**
- Designed dual-stage op-amp cascade (MCP6002) to achieve 1092× gain for microvolt IF signals.
- Implemented 4th-order Butterworth digital filters in Python.
- Wrote firmware on ESP32-S3 to:
  - Sample ADC at correct timing
  - Perform FFT via NumPy (prototyping) then C (optimization)
  - Run MUSIC algorithm for direction estimation
  - Integrate Extended Kalman Filter for tracking
- Debugged cross-layer issues: ADC quantization affected filter stability, tuned virtual ground biasing, tested with oscilloscope.

**Result:** Working phased array radar with <5 degree angular error, real-time tracking.

**Reflection:** Hardware-software debugging requires understanding both layers. When something fails, could be analog, digital, or firmware. Methodical breakpoints help.

---

**Situation 2: Low-Level Systems Thinking (TriCore9 CPU Design)**

**Situation:** Build custom 8-bit RISC processor from scratch. Design ISA, pipeline, assembler, validation.

**Task:** Optimize critical path to minimize clock cycle time while managing pipeline hazards.

**Action:**
- Architected 4-stage pipeline: Fetch, Decode, Execute, Writeback.
- Identified hazards: data hazards (register read/write conflicts), control hazards (jumps).
- Implemented hazard detection logic in SystemVerilog.
- Wrote Python assembler to translate mnemonics to machine code (validates ISA).
- Tested with Hamming code algorithm (complex enough to stress all instructions).
- Simulated in ModelSim, debugged timing violations.

**Result:** Working processor. Validated with non-trivial algorithm. Reduced critical path via hazard stalls.

**Reflection:** Low-level systems require thinking about every bit. Abstractions matter. Small optimizations compound.

---

**Situation 3: Debug Complex Embedded Systems (Mini-Stockfish Neural Network)**

**Situation:** Chess engine neural network inference. Started with 13M position dataset, grew to 316M. Original pipeline broke at scale.

**Task:** Debug performance issues, redesign data pipeline for large datasets.

**Action:**
- Original pipeline: loaded entire 13M dataset into RAM. Worked fine.
- Grew dataset to 316M (24x). Pipeline broke: out of memory.
- Analyzed bottleneck: naive loading strategy doesn't scale.
- Redesigned: memory-mapped files, PyTorch DataLoaders with streaming, async batch loading.
- Benchmarked: training time per epoch went from 2 hours to 45 minutes.
- Validated: model quality maintained despite pipeline rewrite.

**Result:** Successfully trained 316M-example NNUE. Competitive play strength.

**Reflection:** Scaling systems requires rethinking, not just optimizing. Data pipelines are often the bottleneck.
