**Task:** Reduce API latency bottleneck without increasing costs.  
**Action:**
- Identified two-tier caching strategy: Redis (server-side) + request coalescing (client-side)
- Redis cache: 90% latency reduction (Square API calls from 2s+ to <200ms p99)

- Tested and measured before deploying to production

**Result:** p99 latency <200ms. Users experienced instant inventory updates.  
**Reflection:** Good systems thinking means understanding where bottlenecks live (network, cache, query complexity) and matching solutions to constraints.
