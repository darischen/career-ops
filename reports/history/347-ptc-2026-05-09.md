### Story 3: Performance Optimization (Unblocks Product)

**Situation:** Groundwork Books e-commerce (4,000+ SKUs, real-time inventory)

**Task:** Square API latency was killing search experience (2-3s response time). Platform relied on Square for live inventory.

**Action:**
- Profiled requests: discovered 50+ concurrent lookups for single search
- Implemented Redis look-aside cache (reduce API calls by 90%)
- Added client-side request coalescing: batch 50 concurrent calls into 1 API call with 2-minute TTL

**Result:** Search UX went from unusable to instant. User engagement increased 25%. Inventory stayed real-time synchronized without hammering Square API.

**Reflection:** "I learned that backend performance directly enables frontend features. At Arena, if I notice a query is slow or an API is unreliable, I'll raise it early rather than assume it's the backend team's job to fix."
