## Block F — Interview Prep & STAR Stories

**Story 1: CPU Architecture & Optimization (TriCore9)**
- **Situation:** Fall 2025, 4-month course project, custom 8-bit RISC CPU design, SystemVerilog RTL
- **Task:** Design a 4-stage pipeline (Fetch, Decode, Execute, Writeback) minimizing hazard stalls
- **Action:** Built pipeline in SystemVerilog, ran ModelSim simulations, identified critical path bottlenecks (data hazards, control hazards), iterated on pipeline stages, added hazard resolution logic
- **Result:** Functional CPU passing Hamming Code test cases. Pipeline with minimal hazard stalls, optimized critical path.
- **Reflection:** "I know where latency comes from. Pipeline hazards, instruction dependencies, memory bottlenecks—I've debugged all of them. AV software runs on CPUs with the same constraints. This TriCore9 experience is directly applicable to Nuro's performance work."

**Story 2: Microsecond-Level Performance Obsession (Mini-Stockfish)**
- **Situation:** Mar–Mar 2025–2026, chess engine project, leaf node evaluation latency was 1.1ms per position
- **Task:** Achieve competitive play strength while maintaining sub-millisecond responsiveness (real-time constraint)
- **Action:** Profiled execution, identified bottlenecks (move generation, memory allocation, cache misses), optimized critical path: Cython + C++ for move gen, bitboard representation for cache locality, reduced memory allocations in hot loops, added transposition table for reuse
- **Result:** 77.2% latency reduction (1.1ms → 0.25ms), engine now sub-millisecond responsiveness, trained on 316M positions, competitive Elo
- **Reflection:** "Latency obsession is my baseline. Measure, profile, optimize, repeat. No guessing. That's the methodology your Performance team applies to AV stacks. If I see a microsecond being wasted, I want to fix it."

**Story 3: Cross-Layer Understanding (HB100 Phased Array Radar)**
- **Situation:** Feb–Apr 2026, E4E research project, phased array microwave sensor for autonomous detection
- **Task:** Build end-to-end signal processing pipeline: analog front-end → digital sampling → DSP → tracking algorithms
- **Action:** Designed op-amp cascade (MCP6002) for 1092x gain on microvolt IF signals, 4th-order Butterworth filters (analog + digital), FFT, MUSIC algorithms for direction estimation, Extended Kalman Filter to overcome local oscillator drift (hardware limitation)
- **Result:** Sub-degree direction resolution, 2D spatial tracking despite hardware imperfections
- **Reflection:** "I've debugged hardware-software interactions at the signal level. Performance isn't just code—it's how hardware constraints cascade into software choices. AV stacks have these cross-layer dependencies everywhere. I think in those terms."

**Story 4: Pragmatism Under Pressure (FlipperZillow)**
- **Situation:** Feb–Mar 2026, 24-hour hackathon, GPU pipeline for 3D house tours using AMD ROCm
- **Task:** Process property images through DepthAnythingV2 + DFormerV2 + SAM3D within 24-hour deadline
- **Action:** SAM3D broke on ROCm (dependency conflict). Rather than spend 8 hours debugging, cut SAM3D, kept depth + segmentation, shifted to Nvidia cloud with CUDA, shipped working product
- **Result:** Delivered end-to-end 3D tours with professional narration in 24 hours, proved GPU inference pipeline architecture
- **Reflection:** "Performance optimization means making trade-offs. You can't optimize everything. I know when to cut scope, when to switch strategies, when to ship a working product instead of chasing perfect. That judgment is critical for AV teams."

**Story 5: Team Leadership at Scale (PatentIQ)**
- **Situation:** Jan–Mar 2026, 4-month Product Manager Accelerator, 14-member team, hard MVP deadline
- **Task:** Architect and ship AI patent search platform
- **Action:** Designed system architecture (FastAPI + PostgreSQL pgvector), delegated work by skill (data scientist owns ML, backend owns integration, frontend owns UI), tracked progress weekly via ClickUp, unblocked dependencies
- **Result:** Shipped production MVP on time, clear ownership boundaries, minimal rework, 14-person team coordinated
- **Reflection:** "I don't just optimize code; I optimize teams. Clear scope, measured progress, no surprises. That mindset scales to larger performance initiatives."
