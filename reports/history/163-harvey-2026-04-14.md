## F. Interview Prep

**Role archetype:** Full-Stack AI Product Engineer. Daris is strong here. 5 recommended stories below.

### Story 1. End-to-End Product Ship (PatentIQ)
**Situation:** Led 14-person team to ship patent search + drafting assistant in 4 months. Founder-led startup (PMs, designers, developers).
**Task:** Architect and ship full-stack product (backend, frontend, database, auth, deployment).
**Action:** Designed system architecture (FastAPI + Next.js + Supabase). Owned backend development (semantic search, recursive SQL, caching). Integrated Auth0 for user management. Deployed to Vercel (frontend) and EC2 (backend). Led weekly standups, unblocked teammates.
**Result:** Shipped MVP on time. Users immediately found value. System scaled to 1000s of patent searches without latency issues.
**Reflection:** Shipping is a multiplier. Making decisions fast (PostgreSQL over Pinecone) and moving forward is worth more than perfect architecture.

### Story 2. Secure Systems Design (Auth0 + Data Privacy)
**Situation:** PatentIQ handled sensitive patent data. Users trusted us with their IP search history. Security is critical.
**Task:** Design authentication and data privacy. Avoid building custom auth (risky).
**Action:** Integrated Auth0 (industry standard, battle-tested). Enforced HTTPS. Stored user search history encrypted. Implemented user access controls (users can only see their own searches).
**Result:** Zero security incidents. Users trusted the system with confidential IP.
**Reflection:** Security is not a feature, it's a requirement. Use proven tools (Auth0, TLS, encryption). Don't roll custom.

### Story 3. AI Feature Integration (Claude API)
**Situation:** Patent search alone wasn't enough. Users wanted drafting assistance (writing patent claims, descriptions).
**Task:** Integrate Claude API to generate drafting suggestions based on search results.
**Action:** Called Claude API with patent context (top-5 search results + user intent). Streamed responses to frontend (real-time feedback). Added user feedback loop (did this help? yes/no) to improve prompts over time.
**Result:** Drafting feature was 2x more popular than search. Users spent more time in product.
**Reflection:** AI features are powerful but need feedback loops. User signals guide improvement.

### Story 4. Building Under Constraints (FlipperZillow 24-Hour Hackathon)
**Situation:** 3D house tour platform with 5 pre-trained models in 24 hours. Extreme time pressure.
**Task:** Deliver working product despite tight deadline and unfamiliar tech (ROCm, DepthAnythingV2, DFormerV2).
**Action:** Prioritized ruthlessly. Integrated depth estimation + narration. When SAM3D broke on ROCm, cut it. Shipped product without perfect 3D.
**Result:** Working 3D house tours. Won hackathon.
**Reflection:** Constraints force clarity. Shipping a working product beats debugging forever.

### Story 5. Cross-Functional Collaboration (PatentIQ Team Leadership)
**Situation:** 14-person team with diverse roles (PMs, designers, backend, frontend, operations).
**Task:** Unblock team. Make decisions. Move fast.
**Action:** Attended designer standups to understand UI constraints. Talked to PMs about user feedback. Coordinated with operations on deployment. Made architectural decisions collaboratively (PostgreSQL vs Pinecone with product team).
**Result:** Team moved in sync. No silos. Shipped MVP on schedule.
**Reflection:** Engineering is 10% technical, 90% communication. Listening to other disciplines multiplies your impact.
