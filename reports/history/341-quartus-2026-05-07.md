### Story 2: Advocated for Technical Approach (Architecture Judgment)
**Situation:** PatentIQ MVP, choosing between Pinecone (managed) vs PostgreSQL pgvector (self-managed) for semantic search over USPTO dataset.  
**Task:** 1.5M patent documents, 5000-dim OpenAI embeddings, real-time search latency budget <2 seconds.  
**Action:** Benchmarked both approaches. Pinecone was easier but added cost per query + vendor lock-in. PostgreSQL pgvector was self-managed but gave control. Advocated for pgvector with caching layer.  
**Result:** Hybrid ranking (70% semantic, 30% keyword) achieved <2s latency. Pinecone would have cost 10x more at scale.  
**Reflection:** Understanding trade-offs across abstraction layers (database, caching, ranking algorithms) lets you make better architectural decisions. Applied this thinking to hardware: understand constraints at every layer.

### Story 3: Shipped Under Hardware Constraints (Real-World Integration)
**Situation:** HB100 phased array radar, 4-element Doppler system, custom op-amp conditioning, oscillator drift causing navigation errors.  
**Task:** Extract target position and velocity from analog sensor data. MUSIC-based pseudospectrum approach was theoretically perfect but fragile to oscillator drift.  
**Action:** Implemented zone-based amplitude classifier as fallback (monopulse-based angle estimation). Dual-pipeline architecture let us switch strategies based on signal quality.  
**Result:** <5% heading error across multi-target scenarios. Pragmatic trade-off beat perfect but brittle algorithm.  
**Reflection:** Hardware integration requires understanding analog constraints + digital processing. Real-world data is messy. Robustness > theoretical purity.

### Story 4: Led Diverse Team Under Pressure (Team Leadership)
**Situation:** PatentIQ 4-month accelerator, 14-member team (6 engineers, 4 product/design, 4 operations), shipped MVP.  
**Task:** Architect system, deliver foundation layers (API, database, auth), unblock team members on parallel work.  
**Action:** Daily standups (15min), clear API contracts before implementation, pair programming for blocked issues, prioritized vertical slices (login → search → results → draft) over horizontal layers.  
**Result:** Shipped working product. Team shipped first customer-facing features by week 6.  
**Reflection:** Team velocity depends on clarity (contracts) + availability (mentoring blockers) + morale (celebrating wins). Hardware teams will have similar needs.

### Story 5: Systems Thinking Across Abstraction Layers (Technical Depth)
**Situation:** Mini-Stockfish chess engine optimization, HalfKP NNUE neural network inference on CPU was bottleneck (1.1ms per position).  
**Task:** Reduce leaf node evaluation latency to enable deeper search tree.  
**Action:** Profiled bottleneck. Neural network accumulator updates were cache-misses. Implemented incremental updates with O(1) rollback (instead of O(N) recomputation). Added Cython + C++ backend for move generation.  
**Result:** 77% latency reduction (1.1ms → 0.25ms). Competitive play depth increased 3 ply.  
**Reflection:** Understanding constraints across layers (algorithm design → memory access → CPU cache → compiled backend) lets you optimize end-to-end. Hardware integration is similar: understand constraints at firmware/hardware/system levels to make better software decisions.
