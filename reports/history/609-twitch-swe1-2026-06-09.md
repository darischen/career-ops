## F) Interview Prep

### Story 1: Real-Time System Design (Groundwork Books)

**Situation:** Groundwork Books platform. Goal: real-time inventory sync across 4,000+ products while serving thousands of concurrent users.

**Task:** Design architecture. Square API is slow (100-200ms). Can't hit it on every request.

**Action:**
- Implemented two-tier caching: Redis (look-aside) + client-side request coalescing.
- Result: 90% latency reduction, sub-200ms p99, no API rate-limit errors.
- Trade-off: inventory might be 2 minutes stale (acceptable for books).

**Reflection:** Streaming has similar constraints. You serve millions of users, backend services are slow, you need caching and batching. I understand these patterns.
