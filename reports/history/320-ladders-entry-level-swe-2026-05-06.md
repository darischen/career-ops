## Block F: Interview Prep

**5 STAR+R Stories:**

### 1. **Built Semantic Search with Production LLM Integrations (PatentIQ)**
- **Situation:** PatentIQ needed semantic search for 12K+ USPTO patents. Requirement: find relevant patents for early-stage founders researching IP.
- **Task:** Design and ship semantic search layer that returns relevant patents in real-time.
- **Action:** I used OpenAI embeddings (text-embedding-3-small) to vectorize patent descriptions. Stored vectors in PostgreSQL pgvector. Implemented hybrid retrieval: BM25 keyword search + semantic similarity. Built query builder with Zod validation to handle complex filters (patent class, filing date, inventor). Implemented LRU cache (2-minute TTL) to reduce API calls.
- **Result:** Sub-200ms search latency, saved $20K/year vs. Pinecone, system scaled to handle 1000s of concurrent users.
- **Reflection:** This is exactly what Ladders does: integrate LLM APIs into products. Embeddings, vector search, caching, optimization. Core skills.

### 2. **Shipped Full-Stack AI Feature in 6 Weeks (PatentIQ)**
- **Situation:** PatentIQ accelerator: 6-week deadline, build working MVP from scratch, unclear requirements, 14-person team.
- **Task:** Architect system, make technology decisions, coordinate teams, ship.
- **Action:** I proposed stack (FastAPI + Next.js + Supabase + OpenAI APIs). Made tradeoff decisions (pgvector vs. Pinecone, Redis vs. simple caching). Built API, orchestrated frontend/backend integration. Reviewed code, unblocked PMs, adapted as requirements changed.
- **Result:** Shipped working MVP in 6 weeks. Founder-ready for user research.
- **Reflection:** Ladders is shipping fast too. This story shows I can ship full-stack features under deadline.

### 3. **Integrated Multiple LLMs and APIs (Simplerfy + FlipperZillow + RedShift)**
- **Situation:** Multiple projects needed LLM integration: Simplerfy (Claude API for resume optimization), FlipperZillow (Claude Sonnet for house narration), RedShift (testing 7+ model architectures).
- **Task:** Build robust integration that handles API failures, rate limits, cost optimization.
- **Action:** For Simplerfy, I integrated Claude API to evaluate job descriptions and optimize resumes. Handled API errors, rate limiting, cost tracking. For FlipperZillow, I piped spatial analysis (DepthAnythingV2, DFormerV2) into Claude Sonnet for AI narration. For RedShift, I tested adversarial prompts against ChatGPT, Llama, Vicuna, DeBERTa, DeepSeek, Grok — integrated 7+ models into evaluation framework.
- **Result:** All three projects shipped successfully. Learned what works and what doesn't with different APIs.
- **Reflection:** LLM integration is now core to every product. I have hands-on experience across multiple models and use cases.

### 4. **Moved Fast and Shipped (FlipperZillow Hackathon)**
- **Situation:** 24-hour hackathon. Build 3D house tour platform with AI narration.
- **Task:** Learn unfamiliar stack, integrate 5+ models, ship working product in 24 hours.
- **Action:** I learned ROCm, DepthAnythingV2, DFormerV2, SAM3D in 24 hours. Debugged GPU compatibility. Integrated 4 models (cut SAM3D when it broke). Shipped working product with Vision Pro support and Three.js fallback.
- **Result:** Working app, hackathon finalist.
- **Reflection:** Under deadline, you prioritize ruthlessly. You ship 80% working over 100% perfect. This is Ladders' culture.

### 5. **Optimized System Performance (Groundwork Books Inventory)**
- **Situation:** Groundwork Books e-commerce site was slow due to repeated Square API calls for inventory lookups.
- **Task:** Optimize latency without sacrificing accuracy.
- **Action:** I analyzed the bottleneck (N+1 API calls). Implemented two-tier caching: Redis (reduces 2.1s to sub-200ms) + client-side request coalescing (batch concurrent requests, 2-minute TTL). Tested cache hit rate, verified accuracy.
- **Result:** p99 latency 2.1s → 380ms. 90% cache hit rate.
- **Reflection:** Ladders' products need to feel fast. This story shows I understand latency optimization and user experience.
