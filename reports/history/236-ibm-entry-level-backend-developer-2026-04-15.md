## F. Interview Prep

**Technical focus:**
1. **PatentIQ backend architecture.**
   - Why FastAPI over Django or Flask?
   - Database design (PostgreSQL schema, indexing, optimization).
   - API design decisions (REST conventions, error handling, versioning).
   - Why pgvector for embeddings? Trade-offs vs. external vector DB.

2. **Production systems thinking.**
   - How do you approach scaling a backend system?
   - Caching strategies (in-memory, Redis, database-level).
   - Monitoring and observability. How do you detect issues in production?
   - Groundwork's Square API integration — reliability and error handling.

3. **DevOps familiarity.**
   - Git workflows (branching, merging, CI/CD).
   - Containerization (Docker). Have you built Dockerfiles?
   - Cloud deployment (Vercel, AWS, etc.). How do you think about infrastructure?

**Behavioral focus:**
1. **Production mindset.** Emphasize that all projects shipped to real users. You understand production constraints.
2. **Systems thinking.** Discuss how you balance performance, maintainability, and shipping speed.
3. **Learning velocity.** Multiple shipped products show you can learn new stacks quickly.

**Likely questions:**
- Describe your most complex backend project.
- How do you approach API design?
- Tell me about a time you had to optimize a slow system.
- How do you think about database schema design?
- What's your approach to error handling and reliability?
- Why backend engineering? Why IBM?
