## Interview Prep (if you decide to apply)

If you override this recommendation and apply, focus on:

1. **Learning velocity:** Lead with FlipperZillow. Emphasize the 24-hour timeline and how you picked up ROCm, DepthAnythingV2, DFormerV2, and SAM3D from zero.
   - Quote them: "The ramp is steep, the codebase is large, and the feedback loop is short."
   - Your response: "I thrive in that environment. I shipped a working GPU pipeline in 24 hours in an unfamiliar framework."

2. **Codebase navigation:** TriCore9 ISA work and PatentIQ architecture are both large, complex systems you owned. Compilers are large systems too.
   - Highlight the ISA design work (4-stage pipeline, hazard detection, Assembler) as evidence you can reason about compilation and code generation.

3. **Honest gap:** "I haven't written a compiler pass, but I've debugged complex pipelines (GPU kernels, SQL queries, C++ backends). I'm ready to learn IR design and dataflow analysis on the job."
   - Avoid overselling. They value honesty and curiosity.

4. **Research:** Before the interview, read the ONNX / Relay / TVM documentation (Context7 or official docs). Be able to say:
   - "Relay IR lowers ONNX models to target-specific code. Layout selection chooses memory layout (NCHW vs NHWC). Operator splitting decides which ops run on CPU vs GPU."
   - This shows you did homework and can learn fast.

5. **Questions to ask:**
   - "What's the most common compilation failure new hires debug in their first month?"
   - "How do you balance shipping new passes vs maintaining existing ones?"
   - "What's the steepest part of the learning curve for someone without compiler background?"
   - This signals humility and self-awareness.
