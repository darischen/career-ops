## Interview Prep (STAR Stories)

### Story 1: Built a Real-Time Data System (PatentIQ)
**Why this matters for Esri:** Data pipelines, API design, reliability.

**Setup:** "I led PatentIQ's architecture as primary engineer and architect. The challenge was building a semantic search system over 2M+ patent documents that could rank results in <2 seconds while staying cost-efficient."

**Action:**
- Proposed PostgreSQL + pgvector (open-source vector DB) over Pinecone or Weaviate ($ reasons)
- Built recursive SQL query builder with Zod validation for filtering
- Implemented LRU cache to reduce DB latency on repeated queries
- Tested under load: concurrent searches from 14-member team + external users

**Result:** <2s ranking latency, sub-$500/month database costs (vs. thousands for hosted vector DB).

**Why Esri cares:** Shows you can design APIs that scale, think about observability (latency tracking), and make trade-off decisions (cost vs. complexity).

### Story 2: Debugged Performance Under Deadline (Groundwork Books)
**Why this matters for Esri:** Observability, reliability, real-time systems.

**Setup:** "Groundwork was a production e-commerce platform with real-time Square POS sync. We had <200ms p99 latency requirement for inventory lookups."

**Action:**
- Profiled the system: Square API calls were the bottleneck (500ms+ per request)
- Built two-tier cache: Redis look-aside (external) + client-side request coalescing hook (internal)
- Cache hit rate: 95%+ on inventory checks
- Monitored latency in production, iterated on TTL strategy

**Result:** 90% latency reduction (500ms → sub-200ms p99). Real-time inventory across 4,000+ SKUs.

**Why Esri cares:** Shows you can identify bottlenecks in real-time systems, design caching strategies, and operate at scale.

### Story 3: Learned a New Domain Fast (FlipperZillow)
**Why this matters for Esri:** Geospatial is new to you, and this proves you learn domains quickly.

**Setup:** "I built FlipperZillow in a 24-hour hackathon. The challenge: create AI-powered 3D house tours from 2D property photos."
