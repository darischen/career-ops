- **Reflection:** Pragmatism wins. I learned that shipping a working 80% solution beats debugging for 100%. This mindset fits Assort's "hacky and scrappy" ethos perfectly.

### 2. **Architected System from Zero-One (PatentIQ MVP)**
- **Situation:** PatentIQ Product Manager Accelerator. Task: build AI patent search MVP in 6 weeks with 14-person team (PMs, designers, engineers).
- **Task:** Design end-to-end system, lead architecture, serve as primary engineer, ship working product.
- **Action:** I architected the stack (FastAPI backend + Next.js frontend + Supabase database + Auth0 auth). Made critical technical decisions. Designed hybrid semantic search (OpenAI embeddings + PostgreSQL pgvector) for USPTO patent ranking. Built recursive query builder with Zod validation. Implemented LRU cache for latency reduction. Managed team, reviewed code, unblocked PMs.
- **Result:** Shipped working MVP in 6 weeks. Early-stage founders can use it for professional patent research. System designed for future scaling.
- **Reflection:** Zero-one is not just about coding fast. It's about making the right architectural choices early, communicating tradeoffs, and adapting when requirements change. I did that at PatentIQ.

### 3. **Made High-Stakes Technical Decision Under Uncertainty (PatentIQ: pgvector vs. Pinecone)**
- **Situation:** PatentIQ semantic search layer. Two options: managed Pinecone (easier, costs $$$) vs. PostgreSQL pgvector (more ownership, cheaper, but requires custom implementation).
- **Task:** Choose architecture, defend decision, implement.
- **Action:** I analyzed both options: Pinecone trades $20K/year+ for ease, pgvector gives us control and cost savings. Given startup budget and need for customization (recursive filtering), I advocated for pgvector. Designed and implemented the layer myself. Handled edge cases (vector dimension mismatches, query optimization, cache invalidation).
- **Result:** Saved startup ~$20K/year. System performed well under load (sub-200ms query latency on 12K+ patents).
- **Reflection:** Good engineers choose technologies based on tradeoffs, not hype. I can make those calls under ambiguity.

### 4. **Led Cross-Functional Team, Unblocked Non-Technical Stakeholders (PatentIQ)**
- **Situation:** PatentIQ had 14 people: 4 PMs, 3 designers, 4 engineers, 3 ops/support. PMs depended on me for technical feasibility. Designers needed feature specs. Engineers looked to me for direction.
- **Task:** Communicate technical constraints to non-technical team. Unblock roadmap decisions.
- **Action:** Weekly tech sync with PMs to discuss feasibility and tradeoffs. Quick design reviews with designers (accessibility, performance implications). Code reviews for engineers. I didn't tell people what to do. I explained the tradeoffs and helped them decide.
- **Result:** No roadmap delays due to technical misalignment. Team shipped on schedule.
- **Reflection:** In startups, you don't just code. You translate between technical and non-technical worlds. Assort explicitly values this ("world-class engineering team", "collaborate internally").

### 5. **Technical Depth: Semantic Search & Neural Networks (PatentIQ + Mini-Stockfish)**
- **Situation:** PatentIQ needed semantic search. Mini-Stockfish involved training NNUE neural networks on 316M positions.
- **Task:** Design and implement both systems at production quality.
- **Action:** For PatentIQ: I engineered hybrid retrieval combining keyword search (BM25) with semantic similarity (OpenAI embeddings). Ranked results using relevance scoring. For Mini-Stockfish: trained neural network, optimized inference (Cython + C++ legal move gen reduced latency 77%), implemented advanced search algorithms (alpha-beta, killer moves, counter-move heuristics).
- **Result:** PatentIQ semantic search performs well on USPTO dataset. Mini-Stockfish is competitive-strength chess engine.
- **Reflection:** I understand both high-level (ranking, relevance) and low-level (latency, memory, inference) optimization. Assort's "voice/transcription models" is a different domain, but the skills (inference optimization, latency, accuracy) transfer directly.
