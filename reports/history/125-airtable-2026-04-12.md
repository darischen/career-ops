## F) Interview Prep

### 6 STAR Stories Mapped to JD Requirements

#### Story 1: PatentIQ Semantic Search Optimization
**JD Req:** Design and implement high-quality product features that are fast, reliable, and user-friendly

- **Situation:** Lead architect for a patent search MVP. Founders needed fast semantic search to disrupt traditional patent research. Keyword-based search was taking 15 seconds per query — too slow to be useful.
- **Task:** Design and implement a semantic search backend that achieves <2s latency without sacrificing precision.
- **Action:** Designed hybrid ranking system. OpenAI embeddings for semantic similarity (70% weight), keyword matching for precision (30% weight). Implemented pgvector for high-dimensional search. Added Redis LRU cache to reduce database latency. Profiled queries, optimized indexes, ran load tests.
- **Result:** Achieved <2s latency on 5M+ patent dataset. MVP shipped and early-stage founders are using it for real patent research. Feature approved for Series A demo.

#### Story 2: HB100 Radar - Pragmatic Systems Design
**JD Req:** Drive technical decisions and architectural improvements

- **Situation:** 4-element microwave radar system with oscillator drift problem. Initial MUSIC-based phase detection approach (theoretically ideal) failed in practice because free-running Gunn oscillators drifted constantly.
- **Task:** Redesign the signal processing pipeline to handle real-world hardware constraints while maintaining multi-target tracking capability.
- **Action:** Abstracted the problem into two independent processing pipelines running in parallel. Pipeline A: MUSIC (theoretically ideal but fragile). Pipeline B: amplitude-based monopulse (pragmatic, robust to drift). Each pipeline produced independent target estimates; Extended Kalman Filter merged them for final tracking.
- **Result:** System could track 3+ simultaneous targets with <5% heading error despite oscillator drift. Your abstraction meant drift no longer broke the system — it just switched modes. Real-world reliability on commodity hardware.

#### Story 3: Groundwork Books - Full-Stack Architecture
**JD Req:** Work across the stack, from backend services to frontend interfaces

- **Situation:** Community bookstore's e-commerce platform suffering from unpredictable latency. Square POS API calls were taking 500ms+ (p99), killing user experience during inventory checks.
- **Task:** Architect a solution that reduces API latency while maintaining real-time inventory accuracy.
- **Action:** Designed two-tier caching strategy. Backend: Redis look-aside cache for frequent queries (hot inventory items). Frontend: React hook that batches 50+ concurrent inventory lookups into a single API call with 2-minute TTL. Deployed on Vercel with Square API middleware.
- **Result:** 90% latency reduction (500ms p99 to 50ms p99). Patrons can now check inventory in real-time. Site went live with semantic book search (Pinecone) added on top. Real users, shipped product.

#### Story 4: TriCore9 - CPU Architecture Design
**JD Req:** Create reusable and maintainable systems and abstractions

- **Situation:** Design a custom 8-bit RISC processor from scratch. No guidance on ISA design, pipeline depth, or instruction set. Blank canvas for architectural trade-offs.
- **Task:** Make design decisions (R-type vs. J-type instructions, 4-stage vs. 3-stage pipeline, hazard detection strategy) that would make the CPU both performant and easy for future researchers to extend.
- **Action:** Chose 4-stage pipeline (Fetch, Decode, Execute, Writeback) with explicit hazard detection. Trade-off: simpler decode logic vs. more control logic, but predictable stall behavior. Created custom Python assembler for rapid ISA validation. Built SystemVerilog RTL. Created comprehensive test suite.
- **Result:** Final design had a clean ISA, predictable hazard stalls, and clear abstraction layers. Future researchers could reason about performance and extend the CPU. Your architectural choices made it easier for others to build on.

#### Story 5: FlipperZillow - Rapid Prototyping Under Constraints
**JD Req:** Learn systems and best practices through structured onboarding and mentorship

- **Situation:** 48-hour hackathon. Build an AI-powered house tour tool. Unknown constraint: Realtor.com API has no 3D reconstruction. Learning challenge: How to infer 3D structure from 2D photos fast enough to be useful on AMD GPUs (unfamiliar platform).
- **Task:** Learn new tools (ROCm, DepthAnythingV2, DFormerV2, SAM3D) and deliver working prototype in 48 hours.
- **Action:** Orchestrated three parallel deep learning models running on ROCm. DepthAnythingV2 for depth maps, DFormerV2 for semantic segmentation, SAM3D for 3D objects. Built pipeline: Realtor.com photos -> GPU inference -> point cloud generation -> mesh optimization -> .glb export for WebSpatial. Claude API generated professional scripts. ElevenLabs added narration.
- **Result:** Shipped working 3D house tours in 48 hours with zero prior ROCm experience. Judges were impressed. You learned rapidly under pressure and delivered a polished product.

#### Story 6: PatentIQ Team Leadership
**JD Req:** Participate in code reviews and technical discussions

- **Situation:** Lead architect for 14-person cross-functional team (engineers, PMs, designers) in a 4-month accelerator. Your job: ensure the team ships on time without building technical debt.
- **Task:** Drive code quality and team growth through mentorship and architectural guidance.

You walked them through the query plan, showed the slow parts, taught them the principle (don't enumerate cartesian products). They rewrote it and saw 10x speedup on their own benchmarks.
