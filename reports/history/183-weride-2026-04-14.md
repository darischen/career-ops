### 6 Mapped STAR Stories (JD Requirements → Your Projects)

#### Story 1: Real-Time Systems Under Hardware Constraints ⭐ (PRIMARY CASE STUDY)
**JD Requirement:** "Robust systems that must operate 24/7 with minimal failure"  
**Project:** HB100 Phased Array Radar

**S – Situation:**  
Built a 4-element Doppler radar system using HB100 microwave modules. Problem: free-running Gunn oscillators have inherent local oscillator (LO) drift. This breaks phase-coherent direction finding (MUSIC algorithm): without stable phase reference, the pseudospectrum collapses.

**T – Task:**  
Achieve <5% heading error on moving target tracking despite oscillator drift. Can't replace the hardware (cost/schedule), so must design robustness into the DSP pipeline.

**A – Action:**  
Implemented dual-pipeline approach:
1. **Primary (MUSIC-based):** 4th-order Butterworth filters → 1024-point FFT → MUSIC pseudospectrum → Extended Kalman Filter for continuous state tracking [x, y, vx, vy]
2. **Fallback (Monopulse-based):** Amplitude-based angle estimation, zone classifier to overcome LO drift; replaces phase-coherent direction finding with amplitude-based monopulse (robust to phase errors)

Profiled each pipeline: identified critical path (FFT bottleneck) and optimized via SIMD where possible. Designed state machine to switch pipelines based on signal quality metrics.

**R – Result:**  
- Achieved <5% heading error under active oscillator drift
- System ran 24/7 in lab environment without failure
- Demonstrates understanding of: real-time trade-offs (precision vs. robustness), systematic debugging under hardware constraints, multi-algorithm orchestration

**Tie to WeRide Role:**  
"In a vehicle, sensors aren't perfect either: GPS jitter, camera latency, lidar noise. You need to design systems that work reliably *despite* imperfections. This is exactly what I did with the HB100. That's the mentality I bring to your team."

#### Story 3: Distributed Caching Under Concurrent Load
**JD Requirement:** "Design and implement real-time communication service frameworks...enhancing troubleshooting capabilities"  
**Project:** Groundwork Books — Redis Two-Tier Caching

**S – Situation:**  
Groundwork Books (leftist bookstore collective) needed live inventory sync across 4,000+ SKUs. Square POS API is the source of truth; each lookup takes 1-2s. Frontend makes 50+ concurrent requests (checking availability across multiple books). User waits >10s for page load.

**T – Task:**  
Reduce latency to <500ms p99 without duplicating inventory data or losing consistency. Must handle race conditions (inventory updates mid-request).

**A – Action:**  
1. **Look-aside Redis cache:** Set TTL to cache invalidation interval; cache hits bypass Square entirely; reduced 50+ API calls to 0 on cache hits.
2. **Request coalescing:** Client-side hook batches 50+ concurrent requests into single API call; results cached for 2 minutes; subsequent requests read from cache.
3. **Invalidation strategy:** On POS inventory update, invalidate relevant cache keys via webhook.
4. **Profiling:** Measured cache hit rate, miss patterns, request batching efficiency. Tuned TTL based on inventory update frequency.

**R – Result:**  
- Reduced p99 latency from 2s+ to <200ms
- Increased cache hit rate to 85% on common queries
- System stayed responsive under peak load (book launch events)
- Demonstrates: distributed systems reasoning, race condition handling, performance optimization

**Tie to WeRide Role:**

#### Story 5: Embedded Systems & Hardware Integration
**JD Requirement:** "In-vehicle systems development...real-time, robust systems"  
**Project:** HB100 Analog Front-End Design

**S – Situation:**  
4×1 array of HB100 microwave transceivers produce microvolt-scale intermediate frequency (IF) signals. These need to be conditioned for a 12-bit ADC with unipolar input range (0–3.3V). Problem: raw IF signals are bipolar (±few mV); ADC can't digitize negative voltages.

**T – Task:**  
Design analog front-end that amplifies microvolts to ADC range without saturating, handles bipolar IF signals on unipolar ADC, and introduces minimal noise.

**A – Action:**  
1. **Dual-stage op-amp cascade:** MCP6002 configured as 52× non-inverting + 21× inverting amplifier = 1092× total gain.
2. **Virtual ground biasing:** Set 1.65V rail reference at the center of ADC range; maps bipolar IF (±1.65V around virtual ground) to unipolar ADC range (0–3.3V).
3. **EMI filtering:** 5V bypass caps on power rails, pre-amp bandwidth limiting (2nd-order RC filter), output buffering.
4. **Validation:** Tested across temperature range (-10°C to +50°C); measured THD, frequency response, noise floor.

**R – Result:**  
- 1092× gain with <1% THD across operating range
- Stable operation across temperature and humidity extremes
- Successfully digitized microwave transceiver signals for DSP pipeline

**Tie to WeRide Role:**  
"This shows I understand hardware integration: analog signal conditioning, sensor interfaces, debugging at the hardware level. In a vehicle, you'll deal with similar challenges: noisy sensors, operating across extremes, needing stable, low-latency data paths. I've done that."
