### Case Study: HB100 Phased Array Radar

**When to present:** If asked about embedded systems, real-time constraints, or dealing with hardware limitations.

**Framing:**
*"I built a real-time Doppler radar system (HB100 modules, 4-element array). The challenge wasn't just firmware—it was understanding hardware constraints. Oscillators drift, so I couldn't use phase-coherent direction finding like theory says. Instead, I built two parallel DSP pipelines: one using MUSIC (better under ideal conditions), one using monopulse amplitude (robust to drift). System runs 24/7 with <5% error."*

**Why it's relevant to WeRide:**
- In-vehicle systems have real hardware constraints (temperature, vibration, thermal throttling, power)
- Software has to work *despite* hardware imperfection, not assume ideal conditions
- Real-time constraints matter (sensor timestamps, synchronization, latency budgets)
- Dual processing pipelines → reliability thinking (fallback systems)
