### Story 2. Advocated for Technical Approach and Won

**Situation:** PatentIQ MVP architecture decision, 4-month accelerator timeline, data scientist pushing for Pinecone/Weaviate, PMs neutral

**Task:** Choose database + vector storage for semantic patent search system at scale (10M+ USPTO patents)

**Action:**
- Evaluated options: MongoDB/Pinecone (separate vendor, $$), Weaviate (managed service, high cost), PostgreSQL + pgvector (open source, zero cost)
- Argued: "Postgres already running for auth/user data. pgvector is free. MVP doesn't need managed vector DB yet. Easier to iterate on schema."
- Proposed test: Benchmark Postgres + pgvector against Pinecone on 100K patent sample
- Data scientist and backend engineer ran benchmarks independently, came back agreeing within days
- Won technical buy-in without overriding anyone

**Result:**
- Lower infrastructure cost (saved ~$500/month in month 1)
- Faster iteration on search ranking (schema changes in minutes, not hours)
- Later simplified transition to managed vector DB if needed (data already in Postgres)

**Reflection:**
- Technical decisions are persuasion, not dictatorship
- Asking people to validate independently is more powerful than asserting authority

**Why it matters for Giga:** Infrastructure decisions at voice AI scale are costly. Advocating for pragmatic choices (open source first, managed services later) shows systems thinking.

### Story 5. Real-Time Systems Performance

**Situation:** HB100 phased array radar, real-time direction estimation under noisy signal conditions

**Task:** Implement signal processing pipeline for 4-element microwave array to estimate target direction and velocity in real time

**Action:**

**Result:**
- Real-time tracker with sub-degree direction accuracy
- Velocity estimates stable across occlusions
- Hardware proof point on real-time constraints

**Reflection:**
- Latency is a feature, not a bug
- Tradeoffs between accuracy and speed must be explicit

**Why it matters for Giga:** Voice agent systems have hard latency constraints (sub-500ms end-to-end). Hardware background shows deep understanding of real-time pipelines, not just web latency tuning.
