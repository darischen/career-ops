### Story 1. Performance Optimization Under Pressure (Mini-Stockfish)
**Situation:** Chess engine evaluating 1M positions per second needed <250ms latency for interactive play.
**Task:** Profile bottleneck, identify critical path, optimize without rewriting entire engine.
**Action:** Used cProfile to pinpoint leaf node evaluation (77% of CPU). Rewrote evaluation loop in Cython (10x speedup). Rewrote legal move generation in C++ (5x speedup). Implemented Zobrist hashing (2x cache hit rate). Measured every change.
**Result:** 77.2% latency reduction (1100ms to 250ms). Maintained correctness (validated against original engine).
**Reflection:** Performance is measurable. Profiling + incremental optimization beats guessing.

### Story 2. Architecture Design (PatentIQ)
**Situation:** Patent search platform needed to scale from 0 to 100K monthly searches while reducing latency and cost.
**Task:** Design retrieval architecture. Options: managed services (MongoDB + Pinecone) vs open source (PostgreSQL + pgvector).
**Action:** Modeled cost per query for 100K/month. Benchmarked both architectures on USPTO patent dataset (12k docs). Calculated TCO: managed services $3K/month, PostgreSQL $1K/month. Presented to product leads.
**Result:** Chose PostgreSQL + pgvector. Shipped in 2 months. Zero latency issues in production.
**Reflection:** Architecture is about tradeoffs, not absolutes. Understanding cost + correctness + latency guides decisions.

### Story 3. Distributed Computing Thinking (FlipperZillow GPU Pipeline)
**Situation:** 3D house tour platform needed to process 1000s of images through 5 pre-trained models (DepthAnythingV2, DFormerV2, SAM3D) in 24 hours.
**Task:** Design GPU pipeline to parallelize inference without OOM.
**Action:** Used AMD ROCm SSH to distribute inference across GPU cluster. Pipelined model execution (while GPU 1 runs DepthAnything, GPU 2 loads DFormerV2). Implemented memory pooling to reuse allocations. When SAM3D broke on ROCm, cut it and shipped without.
**Result:** Processed 500+ properties in 24 hours. Shipped working product.
**Reflection:** Systems thinking applies everywhere. Pipeline architecture, data flow, resource constraints are universal.

### Story 4. Query Optimization (PostgreSQL pgvector)
**Situation:** Patent semantic search returning 5000+ results, then filtering in application layer. Slow.
**Task:** Push filtering down to database. Build efficient multi-condition WHERE clause.
**Action:** Analyzed query patterns. Built recursive SQL query builder (Zod-validated JSON filters). Indexed on (embedding, inventor, patent_class, date). Implemented approximate nearest neighbor search (pgvector index type).
**Result:** Latency improved from 5s to 200ms (p99). Reduced application memory.
**Reflection:** Databases are query engines. Understanding indexing, execution plans, and statistics is foundational.

### Story 5. Learning New Stacks Fast (FlipperZillow Learning Story)
**Situation:** 24-hour hackathon to build AI house tours. Required AMD ROCm, DepthAnythingV2, DFormerV2, SAM3D. None were familiar.
**Task:** Learn and integrate 5 pre-trained models in 24 hours.
**Action:** Read docs + GitHub while teammates set up GPU cluster. Debugged compatibility issues (DFormerV2 on ROCm had import path problems). Integrated models sequentially. When SAM3D broke, debugged and pivoted to alternative.
**Result:** Shipped working product with depth estimation and 3D reconstruction.
**Reflection:** Learning velocity matters more than prior knowledge. Systematic approach (read docs, test, debug, iterate) works for any stack.
