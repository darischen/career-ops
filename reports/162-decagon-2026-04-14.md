## F. Interview Prep

**Role archetype:** Agentic AI / LLM Engineer. Daris is strong here. 5 recommended stories below.

### Story 1. LLM Safety Evaluation & Evals Pipeline (RedShift)
**Situation:** Test LLM safety across 7 models (ChatGPT, Llama, Vicuna, DeBERTa, DeepSeek, Grok). Manual testing was slow.
**Task:** Build automated red teaming framework to generate adversarial prompts and measure safety across models.
**Action:** Implemented Chain-of-Thought (CoT) injection technique to bypass safety filters. Built PyTorch DataLoader to batch-process 1000s of adversarial prompts. Measured attack success rate for each model.
**Result:** Increased attack coverage by 67% across 7 models. Identified which models were vulnerable to CoT injection.
**Reflection:** LLM safety is measurable. Evals drive iteration. You can't improve what you don't measure.

### Story 2. Multi-Model Integration (FlipperZillow)
**Situation:** 3D house tour platform needed depth estimation, floor estimation, 3D object detection, and AI narration. Required integrating 5 pre-trained models.
**Task:** Design inference pipeline to sequentially call DepthAnythingV2, DFormerV2, SAM3D, and then Claude Sonnet 4.6 for narration.
**Action:** Implemented ROCm SSH pipeline for GPU inference. Pipelined model execution (while GPU 1 runs depth, GPU 2 prepares input for next stage). Debugged model compatibility (SAM3D initially broken on ROCm). When blocked, cut SAM3D and shipped without it.
**Result:** Shipped working 3D house tours with AI narration in 24 hours.
**Reflection:** Pragmatism wins. When integration breaks, cut it and ship working product instead of debugging forever.

### Story 3. Hybrid Semantic Search + LLM (PatentIQ)
**Situation:** Patent search assistant needed to retrieve relevant patents and generate drafting suggestions. Large patent corpus (12k+ docs).
**Task:** Design system combining dense semantic retrieval, keyword filtering, and LLM generation.
**Action:** Built hybrid search (OpenAI embeddings + PostgreSQL pgvector for dense + recursive SQL for keyword filters). Fed top-5 results to Claude API for draft suggestions. Measured eval metrics (relevance, draft quality).
**Result:** Users found drafting assistant 50% faster than manual search. Patent search had <200ms latency.
**Reflection:** The best agent is a hybrid agent. Retrieval + generation + human feedback creates value.

### Story 4. Evaluation-Driven Iteration (Mini-Stockfish + PatentIQ)
**Situation:** Multiple versions of systems (chess engine, patent search). How to know which is better?
**Task:** Define metrics, run evals, iterate based on results.
**Action:** For chess, measured Elo rating (competitive strength). For patents, measured recall (finding relevant patents) and precision (avoiding noise). Ran automated evals after each change.
**Result:** Chess engine reached competitive Elo. Patent search recall 95%, precision 85%.
**Reflection:** Metrics are truth. Evals prevent regressions and guide optimization.

### Story 5. Pragmatic Shipping (PatentIQ Leadership)
**Situation:** 14-person team, 4-month timeline to ship MVP. Scope was ambitious (search + drafting + auth + deployment).
**Task:** Prioritize ruthlessly. Ship working product over perfect architecture.
**Action:** Planned weekly milestones. When SSL certificate generation took too long, used Auth0 instead of rolling custom. When pgvector was uncertain, set up fallback to keyword search. Cut non-MVP features (fine-tuning, custom models).
**Result:** Shipped MVP on time. Users immediately saw value.
**Reflection:** Shipping is a feature. A working product today beats a perfect product never.
