**Story 1: PatentIQ distributed orchestration at scale**

- *Task:* Design system for robust, scalable architecture
- *Action:* Built async job queues (FastAPI + PostgreSQL). Implemented caching (LRU for embeddings). Coordinated across frontend/backend/API layers.
- *Result:* Shipped production system handling concurrent requests from 14-person team. Zero data loss.

**Story 2: Groundwork 90% latency reduction**
- *Situation:* Real-time inventory sync with Square API was bottleneck (1–5s latency). User experience suffering.
- *Task:* Optimize without rewriting from scratch
- *Action:* Analyzed bottleneck (API latency). Implemented look-aside Redis cache. Frontend request coalescing (50+ → 1 API call). Batching strategy.
- *Result:* 90% latency reduction. p99 <200ms. Real-time user experience.
- *Why it matters:* Autonomous driving needs sub-100ms latencies for perception/planning. You've optimized systems to this level.

**Story 3: FishSense ML infrastructure standardization**
- *Situation:* Custom R-CNN pipeline was non-standard. New researchers couldn't ramp quickly.
- *Task:* Standardize Detectron2 + enable reproducibility
- *Action:* Built standardized training pipeline. Reduced setup time by 40%. Documented for new researchers.
- *Result:* Faster onboarding, higher reproducibility, better science.
- *Why it matters:* Autonomous driving needs reproducible ML. Nuro's Data Platform team needs engineers who care about infrastructure quality.

**Story 4: Performance optimization mindset**
- *Situation:* Mini-Stockfish chess engine was slow (1.1ms per move evaluation)
- *Task:* Optimize without sacrificing strength
- *Action:* Profiled. Moved hot code to C++. Implemented Zobrist hashing. Killer moves, LMR, NMP. Cython for Python overhead.
- *Result:* 77% latency reduction (0.25ms per move). Ran 4x faster on same hardware.
- *Why it matters:* Autonomous driving is latency-critical. Perception pipeline must run at 10–100 Hz. You understand how to optimize at this level.
