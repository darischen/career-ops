## F. Interview Prep

**Role archetype:** Embedded / Systems Software Engineer. Daris is strong here (HB100 is excellent project). 5 recommended stories below.

### Story 1. Real-Time Embedded Systems Design (HB100)
**Situation:** Build phased array radar system to track moving targets. Requires <5% measurement error, real-time operation.
**Task:** Design analog front-end, signal conditioning, firmware, DSP pipeline.
**Action:** Built analog front-end (MCP6002 op-amp cascade, 1092x gain, virtual ground biasing for bipolar signals). Implemented firmware (interrupt handling, ADC sampling, data buffering). Designed DSP pipeline (4th-order Butterworth filters, FFT, MUSIC algorithm, Extended Kalman Filter).
**Result:** Achieved <5% measurement error. Real-time tracking of multiple targets.
**Reflection:** Real-time systems require understanding at every layer (analog, firmware, DSP). Latency and correctness are non-negotiable.

### Story 2. Performance Optimization Critical Path (Mini-Stockfish)
**Situation:** Chess engine evaluating 1M positions per second. 1100ms for interactive play was too slow.
**Task:** Profile bottleneck. Optimize critical path without sacrificing correctness.
**Action:** Used cProfile to identify leaf node evaluation as 77% of CPU time. Rewrote evaluation in Cython (10x). Rewrote legal move generation in C++ (5x). Implemented Zobrist hashing (2x cache hit). Measured every change.
**Result:** 77.2% latency reduction (1100ms to 250ms). Maintained correctness (validated against original).
**Reflection:** For latency-critical systems, profiling is non-negotiable. Incremental optimization beats guessing.

### Story 3. Sensor Fusion & Data Processing (HB100 Extended Kalman Filter)
**Situation:** Radar measures range and velocity with noise. Need to fuse multiple measurements to estimate target trajectory.
**Task:** Implement state estimation (Extended Kalman Filter).
**Action:** Designed EKF state space (position, velocity). Implemented prediction (motion model) and update (radar measurement likelihood). Tuned process noise and measurement noise covariances.
**Result:** Smooth trajectory estimation despite noisy measurements. <5% position error.
**Reflection:** Sensor fusion is fundamentals of AV systems. Kalman filtering is proven technique for state estimation under uncertainty.

### Story 4. Firmware & Interrupt Handling (HB100)
**Situation:** Radar needs to sample ADC at 10kHz while processing previous samples and communicating results over UART.
**Task:** Design firmware with interrupts to manage sampling, computation, and communication without race conditions.
**Action:** Set up Timer interrupt for ADC sampling (10kHz). Buffer samples in circular queue. Process samples in main loop. DMA for UART transmission (non-blocking).
**Result:** Stable sampling. No data loss. Clean separation between ISR and main logic.
**Reflection:** Interrupt handling is critical for embedded systems. Understanding priority and latency prevents mysterious bugs.

### Story 5. Pragmatic Shipping (FlipperZillow 24-Hour Hackathon)
**Situation:** 3D house tour platform with 5 pre-trained models in 24 hours. Unfamiliar tech (ROCm, DepthAnythingV2, DFormerV2, SAM3D).
**Task:** Deliver working product despite time pressure.
**Action:** Prioritized ruthlessly. Integrated depth estimation + narration (high-value features). When SAM3D broke on ROCm, cut it instead of debugging forever. Shipped working 3D tours without perfect 3D reconstruction.
**Result:** Working product. Won hackathon.
**Reflection:** In critical systems (like AV), sometimes "good enough" is the right call. You don't ship broken software, but you ship on time.
